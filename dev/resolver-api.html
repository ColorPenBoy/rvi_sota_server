POST /filters

Description

Adds a filter to the database.

URL

/api/v1/filters

Data Parameters

  • name â€” A unique identifying name for the filter, between 2 and 100 alphanumeric characters in length.

  • expression â€” A filter. See the filter syntax documentation.

Success Response

Code: 200

Content:

[
  {
    "expression": "vin_matches \"^12ABC\" AND has_component \"AcmeDVDPlayer\"",
    "name": "myFilter"
  }
]

Error Responses

  • Invalid filter syntax

Code: 400

Content:

{
  "code": "invalid_entity",
  "description": "Predicate failed: (Expression failed to parse)."
}

  • Filter name already exists

Code: 409

Content:

{
  "code": "duplicate_entry",
  "description": "Entry already exists"
}
